# 06-01. WLAN 기본 개념 (802.11 표준)

> CCNA 200-301 무선 네트워크 기초 면접 시나리오

---

### 면접관: "우리 회사에서 새 사무실에 무선 네트워크를 구축하려고 합니다. 802.11 표준에 대해 설명해 주시고, 현재 가장 많이 사용되는 표준은 무엇인지 말씀해 주세요."

네, 802.11은 IEEE에서 정의한 무선 LAN 표준으로, 여러 세대에 걸쳐 발전해 왔습니다. 각 표준의 특징을 정리하면 다음과 같습니다.

| 표준 | Wi-Fi 세대 | 주파수 대역 | 최대 속도 | 채널 폭 | 비고 |
|------|-----------|------------|----------|---------|------|
| 802.11b | - | 2.4GHz | 11Mbps | 22MHz | 초기 보급 표준 |
| 802.11a | - | 5GHz | 54Mbps | 20MHz | 5GHz 최초 사용 |
| 802.11g | - | 2.4GHz | 54Mbps | 20MHz | 802.11b 호환 |
| 802.11n | Wi-Fi 4 | 2.4/5GHz | 600Mbps | 20/40MHz | MIMO 도입 |
| 802.11ac | Wi-Fi 5 | 5GHz | 6.9Gbps | 20/40/80/160MHz | MU-MIMO, 빔포밍 |
| 802.11ax | Wi-Fi 6/6E | 2.4/5/6GHz | 9.6Gbps | 20/40/80/160MHz | OFDMA, BSS Coloring |

현재 기업 환경에서 가장 많이 사용되는 표준은 **802.11ac(Wi-Fi 5)**이며, 신규 구축 시에는 **802.11ax(Wi-Fi 6)**를 권장합니다. Wi-Fi 6는 고밀도 환경에서의 효율성이 크게 향상되었기 때문입니다.

---

### 면접관: "2.4GHz와 5GHz 주파수 대역의 차이점에 대해 좀 더 자세히 설명해 주시겠어요? 어떤 상황에서 어떤 대역을 사용하는 것이 좋을까요?"

두 주파수 대역은 각각 장단점이 뚜렷합니다.

**2.4GHz 대역의 특징:**
- 파장이 길어서 **도달 거리가 깁니다** (벽 등 장애물 투과 능력이 우수)
- 사용 가능한 채널이 적습니다 (비중첩 채널: 1, 6, 11 총 3개)
- 전자레인지, 블루투스 등 다른 기기와 **간섭이 많습니다**
- 최대 전송 속도가 상대적으로 낮습니다

**5GHz 대역의 특징:**
- 파장이 짧아서 도달 거리가 짧지만 **속도가 빠릅니다**
- 사용 가능한 채널이 많아서 간섭이 적습니다 (비중첩 채널 약 24개)
- DFS(Dynamic Frequency Selection) 채널이 존재합니다 (레이더 감지 필요)
- 장애물 투과 능력이 2.4GHz보다 떨어집니다

**6GHz 대역 (Wi-Fi 6E):**
- Wi-Fi 6E에서 새로 추가된 대역입니다
- 더 많은 채널과 더 넓은 채널 폭을 사용할 수 있습니다
- 간섭이 거의 없는 깨끗한 스펙트럼입니다

사무실 환경에서는 일반적으로 **5GHz를 주력으로 사용**하고, 넓은 범위의 커버리지가 필요하거나 IoT 장비 연결에는 2.4GHz를 활용하는 **듀얼 밴드 구성**을 권장합니다.

---

### 면접관: "채널 폭(Channel Width)이 넓어지면 속도가 빨라진다고 하셨는데, 그러면 항상 가장 넓은 채널 폭을 사용하면 되지 않나요?"

좋은 질문입니다. 채널 폭이 넓어지면 속도가 빨라지는 것은 맞지만, **항상 넓은 채널 폭이 좋은 것은 아닙니다.**

채널 폭별 특징은 다음과 같습니다:

| 채널 폭 | 장점 | 단점 |
|---------|------|------|
| 20MHz | 간섭 최소, 안정적 | 속도 낮음 |
| 40MHz | 속도 2배 향상 | 사용 가능 채널 수 감소 |
| 80MHz | 속도 4배 향상 | 5GHz에서만 실용적 |
| 160MHz | 최대 속도 | 채널 가용성 매우 제한 |

넓은 채널 폭의 문제점은:
- **사용 가능한 채널 수가 줄어듭니다** (80MHz 사용 시 5GHz에서도 약 6개 채널만 가능)
- **간섭 가능성이 높아집니다** (더 넓은 주파수 범위를 차지하므로)
- **AP가 많은 고밀도 환경에서는 오히려 성능이 저하**될 수 있습니다

따라서 실무에서는 2.4GHz에서는 **20MHz**, 5GHz에서는 **40MHz 또는 80MHz**를 주로 사용하고, 고밀도 환경에서는 채널 폭을 줄여서 더 많은 AP가 서로 다른 채널을 사용할 수 있도록 설계합니다.

### 면접관: "무선 네트워크에서 CSMA/CA라는 용어를 들어보셨을 텐데, 유선의 CSMA/CD와 어떻게 다른지 설명해 주세요."

네, CSMA/CA와 CSMA/CD는 모두 매체 접근 제어(Media Access Control) 방식이지만, 적용 환경이 다릅니다.

**CSMA/CD (Collision Detection) - 유선 이더넷:**
- 데이터를 전송하면서 동시에 충돌을 **감지(Detection)**합니다
- 충돌이 감지되면 전송을 중단하고 랜덤 시간 후 재전송합니다
- 유선 환경에서는 송신과 수신을 동시에 할 수 있어 충돌 감지가 가능합니다

**CSMA/CA (Collision Avoidance) - 무선:**
- 무선에서는 **Hidden Node Problem** 때문에 충돌을 감지할 수 없습니다
- 따라서 충돌을 **회피(Avoidance)**하는 방식을 사용합니다
- 동작 과정:
  1. 채널이 사용 중인지 확인합니다 (Listen Before Talk)
  2. 채널이 비어있으면 랜덤한 백오프 타이머를 기다립니다
  3. 타이머가 만료되면 데이터를 전송합니다
  4. 수신 측에서 **ACK 프레임**을 보내서 수신 확인을 합니다

추가로, Hidden Node 문제를 해결하기 위해 **RTS/CTS(Request to Send / Clear to Send)** 메커니즘을 선택적으로 사용할 수 있습니다. 송신 전에 RTS를 보내고, AP가 CTS로 응답하면 해당 시간 동안 다른 클라이언트들은 전송을 자제하는 방식입니다.

### 면접관: "무선 네트워크 구조에서 BSS, ESS, SSID 개념을 설명해 주시고, 실제 기업 환경에서는 어떻게 구성되는지 말씀해 주세요."

무선 네트워크의 기본 구성 단위를 설명드리겠습니다.

**BSS (Basic Service Set):**
- 하나의 AP와 그에 연결된 클라이언트들로 구성된 **기본 단위**입니다
- 각 BSS는 고유한 **BSSID**를 가집니다 (보통 AP의 무선 인터페이스 MAC 주소)
- 하나의 AP가 커버하는 영역을 **BSA(Basic Service Area)**라고 합니다

**ESS (Extended Service Set):**
- 여러 BSS가 **유선 네트워크(DS: Distribution System)**로 연결된 구조입니다
- 동일한 SSID를 사용하여 클라이언트가 AP 간 **로밍**할 수 있습니다
- 기업 환경에서는 대부분 ESS 구조를 사용합니다

**SSID (Service Set Identifier):**
- 무선 네트워크의 **이름**입니다 (최대 32자)
- 사용자가 연결할 때 보이는 네트워크 이름입니다
- 하나의 AP에서 **여러 SSID**를 동시에 운영할 수 있습니다 (Multi-SSID)

기업 환경의 일반적인 구성 예시:
- `Corp-WiFi` : 사내 업무용 (802.1X 인증, VLAN 10)
- `Guest-WiFi` : 방문자용 (웹 포털 인증, VLAN 99)
- `IoT-WiFi` : IoT 장비용 (PSK 인증, VLAN 50)

이렇게 SSID별로 다른 VLAN과 보안 정책을 적용하여 네트워크를 분리합니다.

### 면접관: "무선 프레임의 종류에 대해서도 알고 계신가요? 유선 이더넷 프레임과 어떤 차이가 있는지 설명해 주세요."

네, 802.11 무선 프레임은 크게 **세 가지 유형**으로 나뉩니다.

**1. Management Frame (관리 프레임):**
- **Beacon**: AP가 주기적으로 브로드캐스트하여 자신의 존재를 알림
- **Probe Request/Response**: 클라이언트가 AP를 찾기 위해 사용
- **Authentication**: 인증 과정에서 사용
- **Association Request/Response**: AP와 연결을 설정
- **Disassociation/Deauthentication**: 연결 해제

**2. Control Frame (제어 프레임):**
- **ACK**: 프레임 수신 확인
- **RTS/CTS**: 채널 예약 (Hidden Node 문제 해결)
- **CF-End**: 무경합 기간 종료 알림

**3. Data Frame (데이터 프레임):**
- 실제 사용자 데이터를 전달하는 프레임입니다

유선 이더넷 프레임과의 주요 차이점:

| 구분 | 유선 이더넷 | 802.11 무선 |
|------|-----------|------------|
| MAC 주소 필드 | 2개 (Source, Destination) | 최대 4개 (SA, DA, TA, RA) |
| 프레임 확인 | 없음 (CSMA/CD) | ACK 프레임 필요 |
| 프레임 종류 | 데이터 프레임 위주 | 관리/제어/데이터 3종 |
| 헤더 크기 | 14바이트 | 최대 36바이트 |
| 암호화 | 선택적 | 거의 필수 (WPA2/WPA3) |

무선 프레임에 MAC 주소가 최대 4개인 이유는 **DS(Distribution System)**를 경유하는 경우 송신자, 수신자, 송신 AP, 수신 AP 주소가 모두 필요하기 때문입니다.

### 면접관: "마지막으로, Wi-Fi 6(802.11ax)가 기존 Wi-Fi 5(802.11ac)에 비해 어떤 점이 개선되었는지 핵심적인 기술 3가지만 설명해 주세요."

Wi-Fi 6의 핵심 개선 기술 3가지를 말씀드리겠습니다.

**1. OFDMA (Orthogonal Frequency Division Multiple Access):**
- Wi-Fi 5까지는 OFDM을 사용하여 한 번에 **하나의 클라이언트**만 통신했습니다
- Wi-Fi 6의 OFDMA는 채널을 **RU(Resource Unit)**로 나누어 여러 클라이언트가 **동시에 통신**할 수 있습니다
- 특히 작은 패킷을 많이 사용하는 IoT 환경에서 효율이 크게 향상됩니다

**2. BSS Coloring:**
- 고밀도 AP 환경에서 동일 채널을 사용하는 인접 AP의 신호를 구별하기 위한 기술입니다
- 각 BSS에 **색상 번호(1~63)**를 부여합니다
- 같은 채널이라도 다른 색상의 BSS 신호는 **무시**할 수 있어, 불필요한 대기 시간이 줄어듭니다

**3. Target Wake Time (TWT):**
- 클라이언트가 AP와 **통신 스케줄을 미리 협상**하는 기술입니다
- 클라이언트는 지정된 시간에만 깨어나서 통신하고, 나머지 시간에는 **절전 모드**를 유지합니다
- IoT 장비와 배터리 구동 장비의 **배터리 수명이 크게 향상**됩니다

추가로 Wi-Fi 6는 **1024-QAM** 변조 방식을 지원하여 Wi-Fi 5의 256-QAM 대비 약 25%의 처리량 향상을 제공하며, **업링크 MU-MIMO**도 지원하여 여러 클라이언트가 동시에 AP로 데이터를 전송할 수 있습니다.

### 면접관: "잘 설명해 주셨습니다. 그러면 Cisco AP에서 무선 설정을 확인하는 기본적인 명령어를 알고 계신가요?"

네, Cisco AP 또는 WLC에서 사용하는 기본 확인 명령어를 말씀드리겠습니다.

#### 기본 무선 확인 명령어

```
! AP에서 무선 인터페이스 상태 확인
show interfaces dot11Radio 0        # 2.4GHz 라디오 상태
show interfaces dot11Radio 1        # 5GHz 라디오 상태

! 현재 연결된 클라이언트 확인
show dot11 associations

! 무선 설정 요약 확인
show controllers dot11Radio 0

! SSID 설정 확인
show running-config | section dot11

! 채널 및 파워 설정 확인
show controllers dot11Radio 0 | include Channel
```

WLC(Wireless LAN Controller) 환경에서는:

```
! WLC에서 AP 목록 확인
show ap summary

! 특정 AP의 상세 정보
show ap name [AP-NAME] config general

! 클라이언트 목록 확인
show client summary

! 무선 네트워크(WLAN) 설정 확인
show wlan summary
```

이러한 명령어들을 통해 현재 무선 네트워크의 상태, 연결된 클라이언트, 채널 설정 등을 확인하고 문제를 진단할 수 있습니다.

---

> **핵심 정리**: CCNA 레벨에서 무선 네트워크의 기본 개념은 802.11 표준별 특성, 주파수 대역 차이, CSMA/CA 동작 원리, BSS/ESS 구조, 그리고 Wi-Fi 6의 핵심 기술을 이해하는 것이 중요합니다.
